====== 개요 ====== 
모니터링을 하기 위해 에이전트가 설치가 되고 관리화면에서 등록된 WAS의 cpu, 메모리 정보 및 어플리케이션 정보 등이 화면에 표시된다.

\\

====== 기능 설명 ======
모니터링 중요 정보를 한 눈에 파악할 수 있다. 모니터링 전체 정보 중 프레임워크, 어플리케이션, JDBC POOL 정보를 요약하여 현재 모니터링 상태를 알 수 있다.
  
\\

====== 화면 설명 ======
  - 메인화면 : 모니터링 도구의 메인 화면으로, 전체 모니터링 정보의 중요 내용을 실시간으로 보여주는 화면이다.
  - 요약 다운로드 팝업 : 모니터링 도구 메인화면에서 보여진 현재 상태의 모니터링된 정보를 다운로드 하는 화면이다.
  
\\
===== 메인 화면 =====
{{:egovframework:oe2:가이드:메인.jpg|}}

  * 리프레쉬 시간 : 리프레쉬 시간을 설정하여 주기적으로 메인화면을 재조회한다.
  * 리프레쉬(Refresh) : 사용자가 수동적으로 화면을 재조회한다.
  * 로그아웃 : 로그인 된 사용자 정보에 해당한 권한에서 벗어난다. 
  * 다운로드 : 요약 다운로드 팝업을 띄운다.


==== 화면 상세 정보 ====

  * 메인화면은 실시간 화면으로, 조회되는 시간은 메인화면 타이틀 옆에 표시된다. 
  * 서버 별, 컨테이너 별 모니터링 상태를 상태 표시를 하여 사용자에게 제공한다.
  * 프레임워크 모니터링은 서버와 컨테이너를 설정하였더라도 컨테이너가 기동된 경우에 한하여 리스트에 표시되며, 기동되었더라도 모니터링 설정을 off 한 경우 데이터가 off로 표시된다. 
  * 현재 상태로 정상, 비정상(임계치 초과), 비활성화를 상태공 색깔로 표시된다.

==== 상태 표시 ====

  * 정상 : 파란색 공. 현재 상태가 정상인 경우 
     - 프레임워크 모니터링의 경우, CPU,메모리,쓰레드 모두 현재상태가 정상인 경우를 의미함
     - JDBC Pool 모니터링의 경우, JDBC Pool 모니터링 현재상태가 정상인 경우를 의미함
  * 비정상 : 빨간색 공. 프레임워크 모니터링에서 모니터링의 현재 상태가 비정상인 경우 
     - 프레임워크 모니터링의 경우, 당일임계치초과수와 관계없음. CPU,메모리,쓰레드의 중 1개라도 현재 상태가 비정상인 경우를 의미함
     - JDBC Pool 모니터링의 경우, 당일임계치초과수와 관계없음. JDBC Pool의 현재 상태가 비정상인 경우를 의미함
  * 비활성화 : 회색 공. 
    - 프레임워크 모니터링에서 비활성화인 경우는 다음과 같다
      - 설정(어드민>MBean 설정)에서 전체 기동 여부를 off한 경우
       - 설정에서 기동시간 설정범위에 현재시간이 포함되지 않은 경우
       - 임계치를 설정하지 않은 경우
    - 어플리케이션 모니터링에서 비활성화인 경우는 다음과 같다
      - 사용자가 어드민 창에서 서버 정보를 등록 했는데, 모니터링 결과(데이터)가 수집되지 않는 경우
  * 임계치 초과: 노란색 공. 어플리케이션 모니터링에서 임계치를 초과한 건 수가 발생한 경우를 의미함
  * off : 설정(어드민>MBean 설정)에서 특정 모니터링의 기동여부를 off한 경우
  * N/A : 설정(어드민>MBean 설정)에서 특정 모니터링의 기동여부를 on으로 설정했으나, 특수한 상황에 의해 모니터링 결과(데이터)가 수집되지 않는 경우. 여기서의 특수한 상황이란 예를 들어, 모니터링 도구 소스의 라이브러리 컴파일 버젼과 사용되고 있는 WAS의 라이브러리 버젼의 불일치 같은 비정상적 상황을 의미함
