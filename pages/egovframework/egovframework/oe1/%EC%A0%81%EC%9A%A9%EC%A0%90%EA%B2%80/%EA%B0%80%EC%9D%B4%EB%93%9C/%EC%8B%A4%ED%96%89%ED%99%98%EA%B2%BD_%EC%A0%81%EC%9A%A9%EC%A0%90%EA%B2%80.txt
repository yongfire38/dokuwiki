====== 실행환경 점검 도구 소개 ======

실행환경 점검 도구는 기능목록관리의 개발컴포넌트 등록 기능에서 등록된 프로젝트 개발 소스가 전자정부 프레임워크 실행환경 클래스들을 참조하고 있는지 여부를 파악하여 보고서를 출력해주는 기능이다.

===== 실행환경 적용율 보고서 ======

개발 프로젝트를 등록한 경우 실행환경 적용율 보고서는 자동으로 출력되게 되며, 개발 프로젝트가 등록되지 않은 경우 개발 프로젝트를 등록하라는 메세지가 나오게 된다. 실행환경 보고서의 경우 실행환경 적용 여부를 알려주는 메세지와 참조된 실행환경 클래스 별로 참조한 개발 프로젝트 클래스들의 목록을 보여주는 표로 구성된다.

{{:egovframework:oe1:적용점검:가이드:실행환경_적용점검:적용점검_실행환경_실행환경보고서1.png|}}

만약 개발 프로젝트가 실행환경을 참조하지 않은 경우 실행환경이 적용되지 않았다는 결과를 나타낸다.

{{:egovframework:oe1:적용점검:가이드:실행환경_적용점검:적용점검_실행환경_실행환경보고서2.png|}}

엑셀저장 버튼을 누르면, 해당 보고서를 엑셀 파일로 다운로드 할 수 있다.

===== 실행환경 적용율 검사 ======

실행환경 적용율 검사는 해당 개발 프로젝트에 등록된 실행환경 클래스들이 어떤 개발환경 클래스에게 참조되고 있는지, 또 개발환경 클래스들은 어떤 실행환경 클래스들을 참조하고 있는지 상호간의 참조 여부를 파악할 수 있는 도구이다.

{{:egovframework:oe1:적용점검:가이드:실행환경_적용점검:적용점검_실행환경_실행환경검사1.png|}}

좌측은 실행환경 클래스들, 우측은 개발 프로젝트의 클래스들을 의미하며, 좌측의 실행환경 클래스들을 선택하면 우측의 개발환경 클래스들 중 해당 실행환경을 참조하는 클래스는 선을 기준으로 위쪽에, 참조하지 않는 클래스들은 아래쪽에 표시된다. 실행환경 클래스는 복수 선택이 가능하다.

{{:egovframework:oe1:적용점검:가이드:실행환경_적용점검:적용점검_실행환경_실행환경검사2.png|}}

우측의 개발 프로젝트 클래스를 선택할 경우 해당 개발 프로젝트 클래스에 참조된 실행환경 클래스들은 좌측에 하일라이트 된다.
