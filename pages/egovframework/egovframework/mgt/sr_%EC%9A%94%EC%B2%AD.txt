====== 개요 ====== 
로그인한 사용자에게 프로젝트 내에서의 전자정부 표준프레임워크의 도입, 적용, 활용과 관련된 서비스 요청(Service Request:SR) 기능을 제공한다. 

====== 기능 설명 ====== 
  * SR 등록 : 사용자는 [[egovframework:mgt:SR 요청#SR 등록]] 화면을 통하여 전자정부 표준프레임워크에 대한 SR의 상세 내용을 입력하고, 전자정부 표준프레임워크 관리환경 시스템에 해당 SR을 등록한다.
  * SR 수정 : 처리 담당자에 의해 접수되거나 처리되지 않고 있는 요청 단계인 SR 건을 [[egovframework:mgt:SR 요청#SR 수정]] 화면을 통해 수정한다.

====== 화면 설명 ====== 
===== SR 등록 =====
새로운 SR 요청 건을 작성하고 시스템에 등록한다.\\
{{:egovframework:mgt:srregist.png|SR 등록}}
  * **요청제목** : 해당 SR을 간략하게 나타낼 수 있는 제목을 입력한다. 필수입력 항목이다.
  * 요청대상 : SR의 대상이 되는 전자정부 표준프레임워크의 구성요소를 선택한다. 다음의 항목들을 선택할 수 있다.
    * 개발환경 : 전자정부 업무 프로그램의 개발에 필요한 제공 환경으로 화면개발도구, 컴포넌트개발도구, 데이터개발도구, 테스트자동화도구, 코드검사도구 등이 대상임 
    * 실행환경 : 전자정부 사업에서 개발하는 업무 프로그램의 실행에 필요한 공통모듈 등으로 업무 프로그램 개발 시 화면, 서버 프로그램, 데이터 개발 및 표준화가 용이하도록 지원하는 응용프로그램 환경이 대상임
    * 공통컴포넌트 : 전자정부 사업의 정보시스템 구축 시 여러 행정기관이 공통적으로 활용하기 위하여 재사용이 가능하도록 기능위주로 개발한 소프트웨어가 그 대상임
    * 기타 : 위 세가지 항목 이외의 기타 대상
  * **요청유형** : SR을 작성하게 된 동기 또는 목적 유형을 선택하는 것으로 요청내용을 포괄할 수 있는 항목을 골라 선택한다. 필수입력 항목으로, 다음과 같은 항목들을 선택할 수 있다.
    * 단순문의
    * 장애
    * 오류
    * 변경요청
    * 기타
  * 요청자 : SR을 등록하는 사용자의 소속 프로젝트와 사용자 이름이 자동으로 표기된다.
  * 요청일시 : SR을 요청하는 날짜가 현재시각 기준으로 자동으로 입력된다.
  * **요청내용** : 필수입력 항목으로써, SR 요청에 대한 내용을 상세히 입력한다.
  * 첨부파일 : SR 요청에 필요한 파일들을 첨부할 수 있다. 최대 10개까지 첨부가능하다.\\

  * 저장 : 작성한 SR 요청 내용을 시스템에 등록, 저장한다. 저장 후, [[egovframework:mgt:SR 목록]] 화면으로 이동한다.
  * 목록 : 작성 중인 SR 등록을 취소하고 사용자가 속한 프로젝트에서 요청한 SR의 목록을 조회한다. [[egovframework:mgt:SR 목록]] 화면으로 바로 이동한다.


===== SR 수정 =====
기존에 요청한 SR 건을 수정한다.\\
해당 SR 건의 처리 단계가 **__요청__** 단계일 경우, [[egovframework:mgt:SR 목록#SR 상세조회]] 화면에서 **수정** 버튼을 클릭하여 접근할 수 있다.\\
{{:egovframework:mgt:mgt_srm_req_srupdate01.png|SR 수정 화면}}
  * **저장** : 수정한 내용을 시스템에 저장하여 반영한다.
  * 목록 : 수정 중인 사항을 취소하고, [[egovframework:mgt:SR 목록#SR 목록]] 화면으로 돌아간다.
