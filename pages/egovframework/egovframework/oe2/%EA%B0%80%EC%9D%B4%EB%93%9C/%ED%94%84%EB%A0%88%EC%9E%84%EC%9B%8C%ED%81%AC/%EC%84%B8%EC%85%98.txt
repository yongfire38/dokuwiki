====== 개요 ====== 
세션 정보 모니터링 페이지는 현재 세션정보와 일정 기간동안의 세션 추이정보를 제공한다.

세션 정보 모니터링 목록 화면에서는 WAS(Web Application Server)를 통하여 알 수 있는 수치 정보를 확인 할 수 있다.\\
또한 그래프를 통하여 현재 사용중인 세션 수치를 실시간으로 확인가능하다.\\
세션 정보는 어드민의 서버관리화면에서 설정된 JNDI 포트를 통하여 모니터링 대상서버로부터 수집되는 정보이며, 웹 어플리케이션 서버(WAS)에서 정의된 export name을 통하여 데이터가 수집되기 때문에 벤더(vendor)별로 세션정보를 제공하지 않는 WAS도 있다. 따라서 WAS에 따라 세션정보를 제공하는지 안 하는지를 확인하고 세션정보를 제공한다면 해당 WAS의 관리콘솔의 세션 정보 에서도 확인할 수 있다.\\

====== 기능 설명 ======
  - 세션 정보 모니터링 목록 : 세션 수행정보의 상세 수치 정보를 보여준다. 또한 그래프를 통하여 현재 사용중인 세션 수치를 실시간으로 확인 가능하다.
\\

====== 화면 설명 ======

===== 세션 정보 모니터링 목록 =====
세션 정보를 나타내는 화면이다. 검색조건은 서버명과 컨테이너명이다.

{{:egovframework:oe2:가이드:프레임워크:세션2.jpg|}}


  * 검색 : 선택 또는 입력한 검색조건을 기반으로 서비스 목록을 조회한다.
  * 리프레쉬 : 최신 모니터링 정보를 재전송한다.

==== 화면 검색 용어 설명 ====
  * 시간 : 마지막으로 수집된 세션 정보의 수집 시간
  * 현재 세션 수: 현재 활성화 되어 있는 세션 수. Active Session은 그래프에서 사용된 용어
  * 생성된 세션 수 : 현재까지 생성된 세션 수
  * 최대 세션 수 : 최대 세션의 수
  * 종료된 세션 최대 존재 시간 : 종료된 세션 중에서 최대로 존재한 세션의 시간
  * 세션 평균 생존 시간 : 현재까지 생존한 세션들의 평균 시간
  * 거부된 세션 수 : 거부된 세션의 갯수
  * 종료된 세션 수 : 현재까지 종료된 세션의 갯수

==== 화면 상세 정보 ====
  * 세션 정보는 Tomcat(톰켓)과 JBoss(제이보스) 웹 어플리케이션 서버(WAS)에서만 측정되며, Jeus(제우스)와  WebLoic(웹로직)에서는 측정할 수 없다.
  * 컨테이너 별 세션 모니터링 정보를 사용자에게 제공한다.
  * 당일 세션 모니터링 정보가 있는 컨테이너만 화면에 조회된다.
  * JEUS 웹 어플리케이션 서버의 경우, 따로 세션의 정보를 나타내지 않아서 활성화된 쓰레드 수(Active Thread Count)를 세션 수로 측정하였다.
