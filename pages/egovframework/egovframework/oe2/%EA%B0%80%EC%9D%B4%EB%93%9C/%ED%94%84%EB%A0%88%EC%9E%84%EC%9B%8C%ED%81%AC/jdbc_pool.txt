====== 개요 ====== 
JDBC Pool 모니터링 페이지는 모니터링하는 WAS의 JDBC Pool의 현황정보를 제공한다.

JDBC Pool 모니터링 목록 화면은 JDBC Pool의 총 개수, 활성화된 개수, idel 상태인 개수 정보를 보여주며, JDBC Pool 업무 등록 팝업 화면을 통하여 사용자가 지정한 JDBC Pool 의 업무 명을 등록한다. 
JDBC Pool의 Active Connection(활성화된 커넥션 수)의 추이를 그래프를 통하여 확인할 수 있으며, 모니터링 운영자는 이를 통하여 JDBC 커넥션의 위험 수치를 즉각적으로 확인할 수 있다. 또한 최근 JDBC Pool의 상태를 상세하게 파악할 수 있다.\\
JDBC Pool 모니터링 정보는 어드민에서 설정하는 JNDI 포트를 통하여 얻어오는 정보이며, 웹 어플리케이션 서버(WAS)를 통해서 데이터를 얻어온다. 따라서 해당 WAS 콘솔 상의 JDBC Pool 정보 에서도 확인할 수 있다.\\


JDBC Pool 정보 통계 화면은 JDBC Pool의 모니터링 정보를 통계하고 보여준다. JDBC Pool 모니터링 목록 화면의 그래프는 실시간 JDBC Pool 데이터라면, JDBC Pool 정보 통계 화면의 그래프는 특정 시점의 수치를 분석할 수 있는 데이터를 제공한다.

\\

====== 기능 설명 ======
  - JDBC Pool 모니터링 목록 : JDBC Pool 모니터링 정보 목록을 보여준다.
  - JDBC Pool 업무 등록 팝업 : JDBC Pool 의 업무 명을 등록한다. 
  - JDBC Pool 정보 통계 : JDBC Pool의 모니터링 정보를 통계하고 보여준다.
\\

====== 화면 설명 ======

===== JDBC Pool 모니터링 정보 목록 =====
JDBC Pool 모니터링 정보를 나타내는 화면이다. 검색조건은 서버명이다. 

JDBC Pool의 경우, 다른 프레임워크 모니터링 항목과 다르게 컨테이너 별이 아닌 서버 별로 JDBC Pool을 사용한다. 따라서 하나의 서버에 속한 컨테이너 들은 서로 같은 JDBC Pool을 사용하게 된다.

{{:egovframework:oe2:가이드:프레임워크:jdbc2.jpg|}}


  * 검색 : 선택 또는 입력한 검색조건을 기반으로 모니터링 목록을 조회한다.
  * 리프레쉬 : 최신 모니터링 정보를 재전송한다.
  * 업무등록 : 각 컨테이너 마다 포함되는 JDBC Pool에 대하여 사용자 업무 명을 등록한다.

==== 화면 검색 용어 설명 ====
  * 시간 : 마지막으로 수집된 JDBC Pool 정보의 수집 시간
  * JDBC Pool 명 : 웹 어플리케이션 서버(WAS)에 설정한 JDBC Pool 이름 및 사용자가 설정한 JDBC Pool 이름
=== Tomcat(톰켓)과 JBoss(제이보스) ===
  * IDLE 커넥션 수 : IDLE 한 JDBC 커넥션 수
  * 전체 커넥션 수 : 전체 JDBC 커넥션 수
  * Active Connection : 현재 활성화 된 JDBC 커넥션 수
  * MAX 커넥션 수: 웹 어플리케이션 서버(WAS)상에 설정해 놓은 최대 JDBC 커넥션 수
  * 생성된 커넥션 수: 현재까지 생성된 JDBC 커넥션의 수
  * 파괴된 커넥션 수: 파괴된 JDBC 커넥션의 수
  * 연결 가능한 커넥션 수: 연결 가능한, 사용 가능한 JDBC 커넥션의 수
=== Jeus(제우스) ===
  * IDLE 커넥션 수 : IDLE 한 JDBC 커넥션 수
  * 전체 커넥션 수 : 전체 JDBC 커넥션 수
  * Active Connection : 현재 활성화 된 JDBC 커넥션 수
=== WebLogic(웹로직) ===
  * Active Connection : 현재 활성화 된 JDBC 커넥션 수
  * CAPA : Capacity 용량
  * LEAK 커넥션 수 : Leak이 난 JDBC 커넥션의 수
  * 사용 가능한 커넥션 수 : 현재 사용가능한 JDBC 커넥션의 수
  * 대기 중 커넥션 수 : 대기중에 있는 JDBC 커넥션의 수

==== 화면 상세 정보 ====
  * JDBC Pool 정보에 사용된 용어는 해당 WAS(Web Application Server)에서 사용중인 용어를 번역하여 사용되었다.
  * 서버 별 JDBC Pool 모니터링 정보를 사용자에게 제공한다.
  * 당일 JDBC Pool 모니터링 정보가 있는 서버만 화면에 조회된다.


\\

===== JDBC Pool 업무 팝업 =====
사용자가 지정하는 JDBC Pool 업무명을 등록하는 화면이다. 사용자는 현업에서 쓰이는 JDBC Pool의 업무명을 등록하여 보다 명확하게 정보를 확인할 수 있다.

{{:egovframework:oe2:가이드:프레임워크:jdbcpool팝업.jpg|}}


  * 등록 : 새로운 업무 정보를 등록한다.
  * 취소 : 입력한 정보를 등록하지 않고 팝업을 닫는다.

==== 화면 상세 정보 ====
  * 해당 JDBC Pool이 속한 서버의 모든 JDBC POOL 리스트가 조회되고, 이를 한번에 변경 가능하다.
\\

===== JDBC Pool 정보 통계 =====
JDBC Pool 정보를 통계성으로 보여주는 화면이다. 검색조건은 서버명, 기간이다.

{{:egovframework:oe2:가이드:프레임워크:statsjdbc.jpg|}}

==== 화면 상세 정보 ====
  * 검색 : 서버명, 컨테이너명, 기간구분, 기간 선택 후 검색버튼을 클릭하여 검색조건에 해당하는 챠트를 조회한다. \\
  * 분 : 분 선택 시 60분 이내의 데이터만 조회 가능함 
  * 시 : 시 선택 시 24시간 이내의 데이터만 조회 가능함 
  * 일 : 일 선택 시 60일 이내의 데이터만 조회 가능함 
  * 월 : 월 선택 시 24개월 이내의 데이터만 조회 가능함 

\\
==== 참고사항 ====
  * 선택한 기간 중 데이터가 없는 날짜 및 시간은 표기되지 않음
  * 챠트 X축의 날짜 및 시간의 처음값은 조회한 기간보다 앞선 일자 및 시간으로 표기됨
